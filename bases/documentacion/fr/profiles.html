Les privilèges dans ABCD se regroupent en des catégories:

1. Privilèges simple sur les bases de données et les format s

2. Permisos sobre las funcionalidades de ABCD.

Para proporcionar permisos a los operadores se definen perfiles que hacen referencia tanto las bases de datos como las funcionalidades a las cuales tienen acceso los operadores pertenecientes a cada perfil.

 Creación de perfiles

Use la opción Crear/editar perfiles para definir la permisología a asignar al los usuarios que se agrupan bajo ese perfil.

Se presenta una hoja de entrada donde se solicita la siguiente información:

Un perfil puede contener tantas bases de datos como sea necesario y poseer acceso a uno o más módulos.  En este último caso se presentará en el menú principal un menú para que el operador pueda moverse a través de los módulos seleccionados.

Una vez que construya los perfiles, debe ingresar los operadores y asignarles el perfil correspondiente.

Los perfiles pueden editarse o eliminarse una vez que hayan sido creados. Sin embargo, si un perfil está asignado a un usuario el mismo no podrá eliminarse hasta tanto se eliminen las relaciones de los usuarios hacia ese perfil.

ABCD posee un grupo denominado adm el cual corresponde al administrador del sistema. No está permitido editar o eliminar el perfil de este grupo y al menos un usuario debe estar asignado al mismo. De otra forma se corre con el riesgo de inhabilitar el acceso funciones críticas del sistema.

Si por algún error se eliminara al administrador del sistema, siga el siguiente procedimiento para asignar un administrador provisional para entrar a ABCD y definir el nuevo administrador:

  1. Edite el archivo config.central ubicado en la carpeta central y coloque un valor provisional en los parámetros $adm_login y  $adm_password. Este valor provisional no debe corresponder con ninguna de las identificaciones de usuario ya existentes en la base de datos de control de acceso

  2. Entre a ABCD y provea como identificación del usuario y palabra clave los valores insertados en $adm_login y  $adm_password, respectivamente

  3. ABCD le proveerá acceso con el perfil del administrador del sistema. Vaya entonces a la opción Administración de usuarios y restituya al administrador del sistema.

  4. Borre los valores colocados en las variables $adm_login y  $adm_password del script config.central

Los perfiles de usuario se almacenan en la carpeta par/profiles. En ella existe un archivo para cada perfil creado. El archivo profiles.tab  es interno a ABCD y provee los parámetros que pueden utilizarse para restringir/acceder a las funcionalidades de ABCD. No debe borrarse ya que es la matriz utilizada para la creación/edición de perfiles.


El archivo adm contiene el perfil del administrador del sistema y puede editarse ni borrarse. Por esta razón no se presenta en la lista de perfiles activos.
