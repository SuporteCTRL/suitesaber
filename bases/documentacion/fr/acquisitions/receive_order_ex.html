La commande émise est presentée, en spécifiant la liste des objets qui la compose. Pour chaque objet les données suivantes sont montrées :
  • Référence de l'objet
  • Nombre de copies acquises
  • Prix unitaire de chaque copie
  • Numéro de recommandation de l'endroit où l'acquisition de l'objet est demandée
  • Citation de la référence qui a généré le bon de commande
  • Base de données dans laquelle sera classé (nouveau) ou est classé (copies) l'objet
  • Type d'acquisition : nouveau ou copies
  • Identification de l'objet, dans la cas où le type d'acquisition correspond aux copies
Cette information est obtenue en appliquant receiving.pft dans l'enregistrement situé surbase de données du bon de commande

S'il est nécessaire, il est possible de modifier les valeurs au nombre de copies et le prix unitaire. Cliquez sur next pour traiter l'objet.

quand ce format est construit une recherche est faite dans la base de données de classemment à laquelle appartient l'objet, en utilisant la clé ID de l'objet donnée dans le bon de commande. Pour déterminer si l'objet existe, le format receiving.pft contient la commande suivante :

if s(v500^d)<>'' then f(npost([v500^d],'NC_'v500^c),1,0) fi/fi)

Cela se fait ainsi, si l'objet qui est en train d'être traité contient l'information de classement de la base de données  et le numéro de contrôle en provenance de l'enregistrement auquel appartient les copies acquises, alors, correspondant "postings" sont situés dans la base de données de classement et si le résultat n'est pas zéro, alors, il est supposé que l'objet existe déjà et le lien est fourni pour sa révision avant de continuer avec le traitement.
