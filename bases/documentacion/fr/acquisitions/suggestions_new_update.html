Création d'une nouvelle recommandation

Les recomendacions qui entrent dans la base sont numérotés par séquence pour affecter la créating des relations avec un autre module d'acquisition de la base de donées. Cette numérotation est faite à travers le fichier control_number.cn situé dans le dossier bases/suggestions/data.

Si vous travaillez dans un système de plusieurs usagers, il faut éviter que deux postes de travail génèrent le même numéro. Pour ce faire, faites procédez de la façon suivante :

  1. Vérifiez si le fichier control_number.cn dispose ou pas la propriété lecture seule
  2. S'il ne dispose pas de la propriété lecture seule alors :
    • le fichier est protégé de sorte qu'un autre poste de travail ne peut le lire
    • le dernier numéro séquentiel attribué est lu et ajouté 1
    • une copie du fichier control_number.cn est faite ( control_number.bak)
    • le numéro séquentiel est ajouté au  numéro du champ du contrôle de l'enregistrement
    • le dernier numéro suéquentiel attribué est enregistré dans le fichier control_number.cn  
    • control_number.cn  le fichier est débloqué de sorte qu'un autre poste de travail peut créer un nouvel enregistrement
  3. Si le fichier control_number.cn possède la propriété lecture seule cette situation est signalée à l'opérateur et un lien est fourni pour essayer à nouveau l'attribution d'un numéro séquentiel

Si après plusieurs tentatives il n'est pas possible d'attribuer le numéro séquentiel, l'Adminsitrateur du Système doit vérifier l'autorisation du fichier control_number.cn et, si nécessaire, modifier les autorisations pour donner les accès.
