Editer le fichier DBN.def pour configurer le fonctionnement d'une base de donnée avec iAH oPAC 

Cette page de configuration contient 4 sections obligatoires pour configurer une base pour IAH:

  • FILE_LOCATION
  • INDEX_DEFINITON
  • APPLY_GIZMO
  • FORMAT_NAME
  • HELP_FORM
  • PREFERENCES

Pour plus d'informations sur les sections en tant que telles, consultez le manuel d'utilisation de l'IAH. Ici, n'est fourni de l'aide que sur l'utilisation de l'utilitaire de configuration dans ABCD, mais pas sur le contenu des sections.

Pour éditer le fichier de configuration, il y a 2 manières :

  1. Utiliser un éditeur de texte (TXT) pour modifier directement le ficher de configuration. Pour cela, cliquez sur le lien Editer en format TXT en haut de la page.
  2. Utilisez le guide fourni par ABCD pour faciliter le remplissage des sections.

Le guide est divisé en 6 sections. Chacune se termine par un lien Ajouter  qui permet d'y ajouter un nouvel élément.

La section   INDEX_DEFINITON permet d'éditer chaque ligne afin de compléter l'information. Pour cela cliquez sur le lien Editer qui accompagne chaque dfinition d'index et qui place les infromations complémentaires dans le formulaire. une fois le formulaire complété, cliquez sur Accepter pour transferer les cellules aux lignes correspondantes de la section INDEX_DEFINITION.

La section FORMAT_NAME  donne accès à l'éditeur de format de la TDC de ABCD pour la création ou la modification des formats correspondants.  On y accède pen cliquant sur le lien Créer modifier format en haut de la section. Lors de la création d'un nouveau format, mettez le dans la section fournissant les informations dans les cellules affichées en cliquant sur Ajouter.

Une fois le formulaire rempli, cliquez sur Valider afin de vérifier la cohérence des informations saisies, puis sur Mise à jour pour sauvegarder le nouveau fichier créé.

Les critères de validation utilisés sont les suivants :

FILE_LOCATION

  1. Vérifiez que tous les noms logiques sont précédés d'un littéral FILE. Sinon, l'ajouter.
  2. Vérifier que tous les chemins sont précédés par le littéral %path_database%. Sinon, l'ajouter.

INDEX_DEFINITION

  1. Vérifiez que tous les noms logiques sont précédés par le littéral INDEX. Sinon, l'ajouter.
  2. Vérifiez qu'au moins un sous-champ contient le nom de l'index.
  3. Vérifiez que les noms logiques des bases des index (^y) ont leur entrées correspondantes dans la section FILE_LOCATION

APPLY_GIZMO

  1. Vérifiez que les noms des fichiers gizmo fournis ont leur entrée correspondante dans la section FILE_LOCATION

PREFERENCES

  1. Vérifiez qu'au moins un formulaire de recherche est actif.
  2. Vérifiez que le nombre de documents par page soit une valeur numérique.
