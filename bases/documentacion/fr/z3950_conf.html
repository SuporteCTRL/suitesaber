Configuration des options pour appliquer le protocole Z39.50 à la procédure de capture des données

Trois options sont présentées :

Servers
Définit la liste des serveurs qui sont présentés dans le menu de sélection sur la page de recherche d'information. Les champs suivants doivent être pourvus :

  • Nom du serveur qui est présenté dans le menu de sélection des serveurs Z39.50
  • URL à uiliser pour leur localisation
  • Port qui est activé pour l'accès au serveur par le protocole Z39.50
  • Nom de la base de données qui sera accessible sur le serveur distant
La base de données des serveurs est stockée dans le dossier bases sous le nom serveurs

Format de conversion

Les données des serveurs Z39.50 sont toujours présentées dans le format MARC. Cette option permet la définition d'un format de conversion par correspondance entre les champs MARC de la base de données Z39.50 et les champs correspondants dans la base de données locale.

La table de conversion se compose de deux colonnes :
  • La première colonne contient l'étiquette des champs qui recevront les données dans la base de données locale
  • La seconde colonne contient le champ dans l'enregistrement MARC pour le transfert des données à la base de données locale
La conversion se produit au moment du transfert d'un enregistrement à la base de données locale.

Les formats de conversion sont stockés dans le dossier /bases/cnv/. La liste des formats existant est appelée z3950.cnv

La table de conversion pour les caractères du MARC-8 à ANSI

Les serveurs Z39.50 représentent les caractères spéciaux en utilisant le codage MARC-8 qui utilise deux positions pour chaque diacritique. Cette table transforme le codage MARC-8 (deux caractères pour chaque diacritique) à ANSI (un caractère pour chaque diacritique).

Si dans une dernière recherche contre un serveur Z39.50 des combinaisons de caractères apparaissent qui ne correspondent pas à la langue correspondante, il est nécessaire d'inclure cette combinaison dans la table de conversion de sorte que le caractère correspondant sera représenté correctement. 

La table de conversion pour les diacritiques est stockée dans le dossier bases sous le nom marc8-to-ansi.tab
