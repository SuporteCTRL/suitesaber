Prêts

Indiquez le code d'utilisateur et le numéro d'inventaire de l'objet que vous souhaitez localiser. Vous pouvez utiliser la Liste pour obtenir une liste alphabétique des utilisateurs ou le nombre de stocks dans la base de données loanobjects. Cliquez sur Prêt pour activer le processus de prêt.

Avec l'aide du nom d'utilisateur:
  1. On peut consulter la base de données des utilisateurs (utilisateurs) et pour le format spécifié dans la configuration pour en extraire les données (loans_usdisp.pft). En outre, les états: le type d'utilisateur et la durée,  si ce dernier paramètre a été défini dans la configuration des politiques.
  2. Il lit la base de données des suspensions et les amendes (suspml) pour vérifier l'utilisateur.
  3. Il lit la base de données de transactions pour déterminer les prêts détenus par l'utilisateur s'il est en retard et la date de retour
  4. Il lit la base de données des réserves afin de déterminer si l'utilisateur dispose d'une copie réservée
Avec le numéro d'inventaire:
  1. Il lit les objets de base de données de l'emprunt (loanobjects) afin de déterminer les caractéristiques de l'objet de fournir le numéro de contrôle et le titre de la base de données où l'objet est catalogué.  Avec ces deux valeurs on lit la base de donnée catalographique pour obtenir l'information du titre dont une copie est en cours de traitement
  2. Il lit la base de données afin de déterminer le statut de réserve du copie en demande
  3. Détermine si l'utilisateur a donné une autre copie du même nom en passant en revue l'inventaire du titre en loanobjects contre l'information obtenue de la base de données de transactions
Il est situé à la mise en oeuvre des politiques fondées sur un type d'utilisateur et le type d'objet qui est fourni. Les règles sont appliquées en tenant compte des paramètres suivants:
  1. La date effective de l'utilisateur
  2. Si vous avez des amendes ou d'une suspension
  3. Le nombre de prêts par l'utilisateur
  4. Si vous avez des prêts en retard
  5. Si l'utilisateur dispose déjà d'un autre exemplaire du même titre
  6. La date effective de l'objet
Si toutes les règles de la politique, il est estimé sur la date d'échéance du calendrier des jours fériés et le prêt est attribué à l'utilisateur, en faisant un enregistrement dans la base de données des transactions (trans).

Le non-respect des règles de la politique alors est délivré le message correspondant et ne sera pas attribué à l'utilisateur la demande de copie.


Renouvellements

Le renouvellement d'un prêt peut être faite  lors du déploiement de l'état d'esprit de l'utilisateur ou de l'utilisation de l'option prévue à cet effet dans le menu module mouvement.
  • Grâce à l'état de compte d'utilisateur
    Sélectionnez le prêt que vous souhaitez renouveler et cliques sur Renouveler. ABCD va examiner si la politique accepte les renouvellements (sur la base du type d'utilisateur et le type d'objet) et, si oui, combien sont admis. Si l'essai est de changer les dates de prêt et de rembourser le prêt. Dans la base de données des transactions on passe au champ 200 les dates actuels du prêt et du remboursement, afin de maintenir l'histoire de l'emprunt et pour la compilation de statistiques
  • Utilisation de l'option de Renouvellement
    Indiquez le numéro de l'inventaire que vous souhaitez renouveler le prêt. Avec cette information ABCD effectue une recherche sur la base des transactions pour obtenir les données du prêt,  le code de l'utilisateur, et choisir la politique à appliquer pour le processus de renouvellement. Une fois ces valeurs obtenues on fait la même chose que dans l'étape précédente.

Retours

Le remboursement d'un prêt peut être effectué par l'utilisateur du compte ou en utilisant l'option à cet effect dans les menus présentés dans le module de circulation (mouvement des prêts)
  • Grâce à l'état de compte d' utilisateur
    Sélectionnez le ñrêt que vous souhaitez restaurer et cliquez sur Restaurer. Avec des informations pour l'utilisateur, le type d'utilisateur et le type d'objet, ABCD détermine la politique de l'objet et s'il y a retard dans le retour. Si c'est le cas pourrait apporter une amende ou de suspendre l'utilisateur, comme prévu dans les reglaments. Les suspensions et les amendes sont stockées dans la base de données suspml.
  • Utilisation de l'option Retourner
    Indiquez le numéro d'inventaire de l'emprunt à rembourser. Avec cette information ABCD effectue une recherche sur la base de transactions pour obtenir les données du prêt, le code de l'utilisateur et de choisir la politique à appliquer pour le processus de retour. Une fois ces valeurs obtenues, on fait la même chose que dans l'étape précédente.
