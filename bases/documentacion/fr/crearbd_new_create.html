Résultats de la création d'une nouvelle base de données

La structure pour stocker la nouvelle base de données est créée dans le répertoire bases.

Les répertoires créés sont les suivants :

  • bases/dbn/aides

  • bases/dbn/data

  • bases/dbn/def

  • bases/dbn/pfts

  • bases/dbn/cnv

A l'intérieur de chacun de ces répertoires, exceptés les dossiers data et cnv, il a été créé le même nombre de sous-dossiers qu'il y a de langues choisies ou sélectionnées. La liste des langues peut être obtenue grâce au fichier lang.tab située dans le répertoire bases.

Le fichier dbn.par est créé dans le répertoire bases/par et le fichier bases.dat est mis à jour, ajoutant la base nouvellement créée.

Une fois le processus de création terminé, vous devez assigner les permissions pour les utlisateurs qui administreront la base de données. Sans cela, elle sera uniquement accessible à l'administrateur système.

Pour remplir la base de données, vous devez disposez d'un fichier ISO avec les données à importer dans la nouvelle structure.

Si le fichier provient de WinIsis or d'IsisDos, il sera nécessaire de convertir le jeu de caractères d'ASCII en ANSI. Cette tâche peut être facilement réalisée par le programme MX disponible dans la bibliothèque de fonctions de CISIS.

Pour cela, lancez MX dans une fenêtre DOS depuis le dossier où se trouve la base de données que vous souhaitez convertir. Utilisez la ligne de commande suivante :

mx dbn iso=dbn.iso convert=ansi -all now tell=100

Le fichier ISO obtenu doit être téléchargé dans le dossier data de la base de données qui vient d'être créée. Pour ajouter les enregistrements, utilisez de nouveau le programme MX, avec la ligne de commande suivante :

mx iso=dbn.iso create=dbn -all now tell=100.

Après avoir créé la base de données, exécuter à nouveau MX afin de générer le fichier inversé, en utilisation la ligne de commande suivante :

mx dbn fst=@dbn.fst uctab=ansi actab=ansi fullinv=dbn -all now tell=100

Note : les commandes de MX sont montrées ici à des fins d'exemple. Vous devriez consulter le manuel de référence de la bibliothèque CISIS.

Si le nombre d'enregistrement à importer n'est pas très élevé (par exemple moins de 2000 enregistrements) vous pouvez utilisez l'option Utilitaires de la barre d'outils de l'écran Saisie de données afin d'importer les enregistrements et de générer le fichier inversé.

Dans la mesure où le temps alloué par le serveur Web à chaque processus dépend de chaque installation, le nombre d'enregistrements mentionné précédemment peut varier. Pour cette raison, il est recommandé d'utiliser MX pour ces tâches.
