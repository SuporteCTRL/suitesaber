Types d´enregistrement

Si la structure d´une base de données dispose de différents types d´enregistrement, ils doivent être structurés sous ABCD.

ABCD définit deux façons de faire face aux différents types d'enregistrement:

  1. Pour les structures MARC
  2. Pour les structures no MARC

Types d´enregistrement (structures Marc)

Cette option permet de définir les types de documents qui seront gérés par la base de données quand il utilise le format MARC. ABCD qui désigne una base de données est structurée en format MARC lorsque la définition de la table de données y compris un champ LEADER (LDR). La structure d´un champ LEADER est dans un fichier appelé leader.fdt, celui-ci situé dans le dossier def de la base de données, qui identifie les domaines suivants:

3005 Etat de l´enregistrement
3006 Type d´enregistrement
3007 Niveau bibliographique
3017 Niveau de codage
3018 Description du catalogage

Le type d´enregistrement est obtenu à partir du champ 3006 du Leader et il aide à identifier le format du champ fixe destiné à être utilisé pour la description bibliographique du titre.

Le format du champ fixe associé à chaque type d´enregistrement est stocké dans une structure similaire au FDT et sont presentes dans le processus de saisie des données selon le type de dossier sélectionné dans le Leader.

Pour définir les types de structures dans les notices MARC, procédez comme suit:

  1. Vérifier l´option Type d´enregistrement: marc dans le module de mise à jour les définitions de base de données
  2. Mettez dans les cases marqués etiquette1;identifié comme l´etiquette du champ Leader qui stocke le type d´enregistrement (3006). Laissez la case vide pour l´etiquette2
  3. Dans la colonneFichier de saisie de données placez le nom de la structure qui contient la description du champ fixe correspondant au type d´enregistrement qu´on est en train de décrire. Utilisez le lien modifier á modifier ou créer la structure en question. Ce lien ouvrira l´éditeur du FDT fixe comme champ est gardé dans une structure similaire á la Définition des champs de la Table de la base de données
  4. Placez dans l´Etiquette 1 Valeur la valeur que peut prendre le champ 3006 pour y associer ce type d´enregistrement. Laissez la colonne Etiquette 2 Valeur vide
  5. Placez dans la colonne Type d´enregistrement,Description la description du type de dossier, sera le moment de la création de la sélection du type  
  6. Répétez les étapes 2-6 jusqu´à ce que les types de documents que vous souhaitez définir

À la fin, cliquez sur le bouton Mettre à jour

Les types d´enregistrement sont définies dans le dossier def stockés dans la base de données, dans le cadre de la langue active à la création, sous le nom de typeofrecord.tab

Types d´enregistrment (structures no MARC)

Cette option permet de définir les types de documents qui seront gérés par la base de données quand il n'utilise pas le format MARC.

À ces fins, il est nécessaire pour définir le type de dossier et d'attribuer des champs de la base de données (maximum deux) sauvent des valeurs à être utilisées  pour identifier le type qui correspond à la procédure d'enregistrement de la création ou l'édition.

Pour définir le type d´enregistrement 2 composants:

  1. Identifier le champ (tag) de la base de données est en cours pour obtenir (ou stockée dans le cas d'un nouveau record) le type d'enregistrement de l'information enregistrée. La version actuelle de ABCD permet de mettre en place deux champs dont le contenu va déterminer le type de documents et donc la feuille d'entrée de données à utiliser pour l'entrée ou la modification des enregistrements
  2. 2. Décrire les types de documents sur la base des paramètres suivants: Nom de la table de définition (FMT) qui contient les éléments devant être utilisés pour construire le formulaire d'inscription à la saisie de données
  3. De la valeur des domaines identifiés au paragraphe (1) d'être considérés comme appartenant à ce type de fichier
  4. De la description du type d'enregistrement, qui sera présenté à l'opérateur au moment de l'admission de procéder à la sélection du type correspondant

Pour maintenir la cohérence dans les paramètres de l'ABCD, les champs fixe sont définis par moyen d'un mécanisme qui contient la liste des champs (tags) de la FDT de la base de données qui sont nécessaires pour construire la forme de revenu. Tout comme le tag reste dans les champs, ils héritent automatiquement les caractéristiques décrites dans la base de données FDT.une FMT qui contient la liste des champs (tags) de la  FDT du base de données que sont nécessaires pour construire le format d´entrée. Étant donnée qu´il ne reste que le tag des champs, ceux-ci héritent automatiquement les caractéristiques décrites dans la FDT de la base de données.

Pour assigner les types d´enregistrement il faut définir les fiches d´entrée qui sont liés à chaque enregistrement (voir le format d´entrée de données (FMT).

Si vous définissez un type d´enregistrement pour une base de données qui ne répondent pas à la norme MARC, procédez comme suit:

  1. Mettez dans les cases identifiés comme etiquette1 et etiquette2 les etiquettes des champs à utiliser pour identifier le type d´enregistrement. La deuxième etiquette est facultative
  2. Sélectionnez la colonne Format d´entrée de données que vous utilisez pour entrer dans le type d´enregistrement. Cette fiche d´entrée de données doit être définit à l´avance  avec l´option Créér/ Modifier les fiches d´entrée
  3. Placez la valeur dans le champ etiquette1 valeur pour identifier le type d´enregistrement
  4. Placez la valeur dans le champ etiquette2 valeurpour identifier le type d´enregistrement (optionel)
  5. Placez dans la colonne Type d´enregistrement, Description la description du type d´enregistrement, tel qu´il sera au moment de la création pour la selection du type correspondante
  6. Répétez les étapes 2-5 autant de fois qu´il y ait de types d´enregistrement dans la base de données
  7. À la fin, cliquez sur le bouton Mettre à jour

Les types d´enregistrement sont définies dans le dossier def de la base de données, dans le cadre de la langue active à la création, sous le nom typeofrecord.tab
