Assignation du numéro de contrôle des enregistrements de la base de données


Cette option permet l'assignation automatique d'un numéro séquentiel aux enregistrements de la base de données.

Dans la TDC, le champ de contrôle est défini avec le type "Autoincrement". S'il doit être utilisé pour la gestion de l'inventaire dans un système d'acquisition, alors il devrait être indexé avec le préfixe CN_ dans la FST

Les numéros sont enregistrés dans le fichier control_number.cn qui se trouve dans le dossier data de la base de données correspondante. Le dernier numéro assigné à un enregistrement est stocké dans ce fichier. Pour modifier la séquence d'assignation de ce numéro, utilisez l'option Rétablir le numéro d'inventaire ou de contrôle que vous trouverez sur la page d'accueil du module de catalogage.

Quand l'assignation automatique d'un numéro de contrôle est demandé, une fenêtre apparaît avec une série de MFN auxquels un numéro sera affecté. Notez que si la série de MFN est trop importante, le temps pris pour assigner des numéros peut causer un "time out" sur le serveur et le processus sera alors interrompu.

Dans ce cas, revoyez la sortie de l'action et le dernier MFN traité, et reprenez le processus à partir du MFN suivant.

Dans la mesure où ce processus met à jour tous les enregistrements dans une série donnée, il est recommandé que la saisie et la modification d'enregistrements soit suspendue jusqu'à ce que la mise à jour soit complète.

Il est recommandé que la base de données soit restructurée après l'assignation de numéros de contrôle, la taille en octets ayant pu doubler durant le processus de mise à jour. En sus, il faut aussi générer le fichier inversé.
