Génération automatique des numéros de contrôle des enregistrements de la base de données

Cette option active l'affectation automatique d'un numéro séquentiel aux enregistrements de la base de données.

Dans la TDC, le champ de contrôle est défini avec le type "auto-incrément". Dans la définition du champ, le code qui doit être inclus est indiqué dans la liste inversée ainsi que les formats de présentation et d'extraction correspondants.

Le numéro est issu du fichier control_number.cn enregistré dans le dossier data de la base de données correspondante. Le dernier numéro assigné à un enregistrement est stocké dans ce fichier.

L'information dans le champ du numéro de contrôle ne peut être insérée directement, le numéro d'enregistrement est attribué par une des méthodes suivantes :

  1. Si le champ si est laissé vide lors de la sauvegarde de l'enregistrement, le numéro suivant sera attribué et sauvegardé dans le champ correspondant
  2. Si vous souhaitez attribuer directement un numéro de contrôle, suivez cette procédure :
    • Cliquez sur le lien Attribuer à côté de la zone de texte du champ du numéro de contrôle.
    • Une nouvelle fenêtre sera ouverte qui permettra l'attribution du numéro de contrôle
    • Cliquez surValider pour vérifier si le numéro de contrôle existe ou pas.
    • Si la vérification est positive (càd que le numéro de contrôle n'existe pas), la valeur est affectée au formulaire de saisie.
    • Si la vérification n'est pas positive (càd que le numéro de contrôle existe déjà), l'enregistrement confirme que le numéro existe déjà. Cliquez sur Retour pour attribuer un autre numéro ou sur Fermer pour fermer la fenêtre.
  3. Pour effacer le numéro de contrôle, cliquer sur Attribuer ce qui laisse le numéro de contrôle en place, puis, cliquez sur Envoyer
