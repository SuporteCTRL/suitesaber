Inventory number recovery or database control number recovery


Inventory number

The inventory number is a numerical sequence managed automatically by ABCD on the holdings database. This number is defined in the database FDT as field type autoincrement. and is increased after every register is created, starting with the last value stored in the database.

The last value stored is derived from the control_number.cn file, located in the folder data of the respective database.

This option modifies the value of the last number assigned, in order to restore the registers' numerical sequence.

Control number

The control number is a mandatory field in those bibliographic databases in which copies will be managed through the holdings database, inasmuch as it represents the link field between the item and its copies.

As in the previous case, ABCD will generate this number automatically by means of a numerical sequence. In the same way, the last number assigned is kept in the control_number.cn file, located in the folder data of the respective database.


This option modifies the value of the last number assigned, in order to restore the control number value of the registers.
