Creation of new display formats

This procedure allows you to generate an export format - or to modify one previously generated. The generated formats can be used to display the information to the screen or  to send information directly to a word processor or to a spreadsheet.

To select an existing format, check the list shown under Formats: and click on the format you require. Then click on Edit which will open a window with the selected format. If you know the formatting language Cds/isis, you can edit the format to make changes or adjustments; otherwise, do not modify the format as it will generate errors and you will obtain no results when requesting a listing.

 In the window where the format is deployed, once the modifications are made click on send to send the format to the output file. 

To create a new format click on Create a format and select the fields you want to include in the output from the list. To select fields, double click on the name or highlight them with the help of the "mouse" and then click on the corresponding arrows. You can use the keys [Ctrl] and [Shift] to select the fields. The single arrows transfer the selected fields from one window to the other and the double arrows transfer all the fields.

You can use the up or down buttons to change the position of the selected fields.

When you have finished with your selection click on Generate format and the generated format will be displayed in the window. If you know the formatting language Cds/isis, you can edit the format to make changes or adjustments; otherwise do not modify the format as it will generate errors and you will obtain no results when requesting a listing.

The following possibilities exist for the generation of formats: 

  • Table
  • Paragraph
  • Paragraph (with labels)
  • Columns ( in table)
  • Columns (delimited)

Table

Create a table format, that is, the first column with the label field and the second column with the content. Each field of the database is placed in the first column, and each record is in the second column.

Paragraph

Create a format where each field is placed in a new line. Do not use labels for the fields. The records are separated by a paragraph break (<P>)

Paragraph (with labels)

As above, but each field is preceded with the label of the field in bold.

Columns (in table)

Create a table with all records selected. Each record is represented by a row in the table, and each field is located in a column. In this type of format an input window is presented in order to include the titles of the columns. By default it puts the name of the field in the FDT, but this can be modified.

This format is ideal for exporting to word processors and spreadsheets and to use as a data source in mail merge processes.

Columns (delimited)

This creates a flat file where each record represents a líne. The selected fields are separated with the delimiter | . If a data field is absent, then, the corresponding position is empty, but the delimiter is present. This is in order not to lose the alignment of columns. In this type of format an input window is presented in order to include the titles of the columns. By default it puts the name of the field in the FDT, but this can be modified.

The results of the implementation of this format can be saved as a TXT file, which can then be processed by an editing program to convert it into a table.

Editing subfields

During the process of generating formats the following measures for the management of subfields are taken.

  • If in the descripton of the Definition Table of fields no information was provided in column 7 (pre-literal), then the subfield delimiters are removed and a blank space is inserted to separate the different components of the field
  • If pre-literal information was contributed, then each subfield is preceded by the corresponding pre-literal. E.g.: if the FDT specifies abc as subfields and a column is specifed in the pre-literal -,. (hyphen comma full stop),  ^a is not preceded with a delimiter (this is the function of the hyphen, the presence of which is mandatory in order to maintain the spacing between delimiters and characters during editing. Unfortunately you cannot use a space as the FDT editor removes spaces at the top and bottom of each  column); ^b is preceded by a comma and ^c is preceded with a full stop 

Once completed with the selection/editing/generation of a format, you can Generate the output.

The records can be selected in two ways:

  1. Through a range of records (from - to)
  2. By applying a search expression to the data base. In order to do this click on the search icon and construct the search expression.

The following options can be used to conduct a search:

  1. Type the search expression directly into the search box
  2. Use the advanced search form to build a search expression.  To do this click on the link New
  3. To construct a search expression from previously stored searches, use the menu that is presented for this purpose

Once you have constructed the search expression you can view the results through the link Preview, and if considered suitable, this can be stored by using the link Save search expression using the link pfts The new search will be stored in the file search_expr.tab located in the sub-folder pfts/language-active of the selected database.

Classification of the obtained results

Prior to displaying the results a classification format can be obtained in order to present/display the records in a defined order. This is by:

  1. Providing a search format directly
  2. Selecting a search format from a list of formats already defined

The formats are defined as:  MPU,v1,v2,MPL, or by utilizing any valid command of the CDS/Isis format language. Note that this only provides the allocation of a search format. If you want to sort by various fields, build the search format fields required.

This option is an additional process for the server and if too many records are requested (depending on the speed of the server and the configuration of the time out limit assigned to the processes in the PHP), the process could be cancelled by "time limit exceeded"

Once the selection has been carried out, click on Send to a word processor (to send the obtained results to a word processor), Send to a spreadsheet, or click on Preview to display the result on the screen.

To display the results click on Generate results and arrange the information according to the method that you will use for the selection of records, either by range of Mfn or by searching the database.

To save the format click on Save format and give a name (only using letters and numbers and no blank spaces) and provide a description to display in the list of Formats already defined.
