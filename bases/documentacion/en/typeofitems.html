Item types

This table describes the policies to apply for the different item types existing in the database. The item type referred to in the table should be generated by the extraction format that is applied to the database for the items to be loaned; this output, amongst other data, should contain the item type to be located in the table for assigning the respective policy.

If the extraction format generates no output then the loans operator will decide the type of loan to apply by selecting one of the options from the table Loan types.

When the record of the loan is made, the item type or loan type that has been assigned is stored to ensure the correct application of the policies for returns, fines and suspension.

Description of the table

Field Use in the application of the policy
Item type Value that derives from the extraction format aplied to the bibliographic database
Description Name of the item type
Type of user Specifies the type of user to which the policy for this item type applies. If no user is selected, the policy applies to all users
Loan period Number of days/hours for calculating the loan period
Unit Unit of measurement for the loan period (Days/Hours)
No. of reservations Number of reservations permitted
Fine per day overdue Shows the fine to be charged for each day/hour overdue in the return of a loan 
Days of suspension per day overdue Indicates how many days the user should be suspended for each day/hour overdue in the return
Reserved. Fine per day overdue If the item is reserved, specifies the fine to be applied per day overdue in the return
Reserved. Suspension per day overdue If the item is reserved, specified the days of suspension to be applied per day overdue in the return
