Automatic generation of database record control numbers

This option enables the automatic assignment of a sequential number to the database records.

In the FDT the control field is defined as field type "Autoincrement". In the definition of the field the code which must be included is indicated in the inverted list and the corresponding display and extraction formats.

The number is taken from the file control_number.cn stored in the folder data from the corresponding data base. The last number assigned to a record is stored in this file.

The information in the control number field cannot be inserted directly, the record number is allocated by one of the following methods :

  1. If the target field is left blank at the time of saving the record, the next number will be assigned and recorded in the corresponding field
  2. If you wish to assign a control number directly follow this procedure:
    • Click on the link To assign next to the text box for the control number field.
    • A new window opens which allows the control number to be assigned
    • Click on Send to validate whether a control number exists or not.
    • If the validation is successful (i.e. the control number does not exist), the value is assigned to the entry form
    • If the validation is not successful (i.e. the control number exists) the record confirms that it is already assigned. Click on Return to assign another number or Close to close the window
  3. In order to erase the control number, click on Assign this leaves the control number in target, thereafter click on Send
