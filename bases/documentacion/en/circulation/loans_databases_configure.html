Required parameters to set up databse with lendable items

Parámetro
Ejemplo
 
 

Pft para desplegar la información bibliográfica del objeto (loans_display.pft)
V50^a,” “v245^a,” “d260,v260^a, …

Pft para almacenar la información del título en la base de datos de transacciones (para efectos de visualización y estadísticas)

(loans_store.pft)
 `^a`,v250^a,’^b’,v245^a,’^c’v260^c ….

   La idea de este formato es construir un campo con subcampos que contenga la información bibliográfica que va a ser almacenada como parte de la transacción del préstamo. El resultado producido por el formato se almacena en el campo 100 del archivo de transacciones de préstamo (trans)

Pft para extraer la información desde la base de datos de transacciones (loans_show.pft)
Mhl, v100,

Por implementar:

Si el registro bibliográfico contiene las existencias, esto es, no se trabaja con loanobject, se requieren adicionalmente los siguientes parámetros:

 
 

Prefijo a utilizar para localizar un número de inventario (loans_conf.tab)
NI=

Prefijo a utilizar para localizar un número de clasificación (loans_conf.tab)
SN_

Pft para extraer el número de clasificación (loans_conf_tab)
V50^a

Tag del campo con las existencias (loans_conf.tab)
900

Formato para extraer el tipo de material

If iocc=1 then ‘R’ else ‘L’ fi

El resultado de este formato debe ser uno de los tipos de objeto considerados en la política de préstamos
