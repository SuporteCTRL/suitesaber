Editar o ficheiro DBN.def para configurar uma base de dados através do IAH 

Esta página de configuração consta de 4 secções necessárias para o IAH configurar uma base de dados:

  • FILE_LOCATION
  • INDEX_DEFINITON
  • APPLY_GIZMO
  • FORMAT_NAME
  • HELP_FORM
  • PREFERENCES

Para informação específica relativa às secções deverá ser consultado o Manual do Utilizador do IAH. Nesta página apenas se fornece uma ajuda sobre a forma de utilizar a ferramenta de configuração prevista no ABCD e não sobre o conteúdo dessas secções.

Para editar o ficheiro de configuração são possíveis 2 opções:

  1. Usar um editor TXT para modificar directamente o ficheiro de configuração. Para tal clicar sobre a ligação Editar no formato TXT que é apresentado no início da página.
  2. Usar o assistente fornecido pelo ABCD para facilitar o preenchimento das secções.

O asistente está dividido nas 6 secções exigidas. Cada uma possue no final uma ligação agregar  para permitir adicionar um novo elemento da secção em alteração.

A secção  INDEX_DEFINITON permite editar cada linha com vista a facilitar o preenchimento da informação. Para tal clicar sobre a ligação editar que acompanha a definição de cada índice e no formulário apresentado forneça a informação correspondente. Uma vez preenchido o formulário clicar sobre Aceitar para transferir as caixas da linha correspondente da secção INDEX_DEFINITION.

A secção FORMAT_NAME  facilita o acesso ao editor dos formatos de descrição (PFT`s) do ABCD para que seja possivel criar ou alterar o formato correspondente.  Este acceso é feito através da ligação Criar/editar formato incluido no inicio da secção. Se tiver criado um novo formato, deverá adicionar o mesmo à secção preenchendo as caixas que são apresentadas depois de clicar em Agregar.

Uma vez preenchida toda a informação no formulário, clicar sobre Validar para verificar a consistência da informação fornecida e depois sobre Actualizar para guardar o ficheiro criado.

Os critérios de validação utilizados são os seguintes:

FILE_LOCATION

  1. Verificação de que todos os nomes lógicos estão precedidos pelo literal FILE. Em caso de falta deste literal o mesmo será adicionado.
  2. Verificação de que todos os caminhos estão precedidos pelo literal %path_database%. Em caso de falta deste literal o mesmo será adicionado.

INDEX_DEFINITION

  1. Verificação de que todos os nomes lógicos estão precedidos pelo literal INDEX. Em caso de falta deste literal o mesmo será adicionado.
  2. Verificação de que está presente pelo menos um sub-campo com o nome do indíce
  3. Verificação de que os nomes lógicos das bases de dados de índices (^y) tenham uma entrada correspondente na secção FILE_LOCATION

APPLY_GIZMO

  1. Verificação de que os nomes dos gizmo fornecidos tenham uma entrada correspondentes na secção FILE_LOCATION

PREFERENCES

  1. Verificação de que está activado pelo menos um formulario de pesquisa.
  2. Verificação de que o número de documentos por página é um valor numérico
