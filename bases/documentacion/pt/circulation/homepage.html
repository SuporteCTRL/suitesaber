Este módulo trabalhaa com as seguintes bases de dados:

users
Base de dados de usuários
loanobjects

Base de dados com as diferentes cópias dos objetos que estão disponíveis para empréstimo.

Existe uma única base de dados loanobjects que reflete o estoque de todos os catálogos definidos no ABCD. Cada cópia é identificada através de um número único ou número de inventário. Além disso. armazena o número de controle do objeto, o volume e o tomo da cópia para estabelecer as diferenças entre as cópias de um mesmo objeto; o nome da base de dados onde está catalogado para identificação da cópia; e o tipo de objeto atribuído à cópia para efeitos de aplicação da política de empréstimos.

Cada objeto representa um registro da base de dados loanobjects e as cópias do mesmo são armazenadas como ocorrências de um campo repetitivo.

Esta base de dados é atualizada a partir da catalogação, onde é permitido transferir os registros de inventário da base de dados copies para a base de dados loanobjetcs.

O menu principal do modulo de Circulação e Empréstimo contém as seguintes seções:

Transacciones

Esta seção define as transações disponíveis:
  • Emprestar
    Através do código do usuário e do número de inventário se acessa a base de dados users e loanobjects para determinar o tipo de usuário e o tipo de objeto. Além disso, é verificado se o usuário tem multas ou suspensões pendentes realizando uma busca na base de dados suspml, ou se tem empréstimos pendentes, realizando uma busca na base de dados trans. Com estes valores é determinada a política de empréstimos a aplicar e calculada a data de devolução
  • Devolver
    É utilizado o número de inventário para localizar na base de dados trans a cópia que se deseja devolver. Se há atraso na data de devolução, é calculada a sanção que é indicada pela política de empréstimos e a mesma é salva na base de dados suspml
  • Renovar
    É utilizado o número de inventário para localizar na base de dados trans a cópia que se deseja renovar.     Além disso, é verificado se o usuário tem pendentes multas e/ou suspensões realizando uma busca na base de dados suspml, ou se tem outros empréstimos pendentes, realizando uma busca na base de dados trans. Com estes valores se determina a política de renovação a aplicar e se calcula a nova data de devolução
  • Suspender/Multar
    Indica-se o código do usuário que se deseja suspender/multar e os dados correspondentes.
  • Estado da conta de um usuário
    Exibe a listagem de empréstimos, suspensões e/ou multas pendentes para um usuário
  • Situación de un objeto
    Exibe todas as cópias de um objeto, indicando aquelas que estão em poder de algum usuário
