  2. Faz uma pesquisa não banco de dados copies para checar se os objetos da presente nota de encomenda foram atualizados. Isso constrói uma expressão de pesquisa com os seguintes campos:

    ORDER_order_suggestion_bidding

    onde order=número da ordem de compra, suggestion=número da sugestão, bidding=número da licitação.

    Esta pesquisa vai tentar evitar a duplicação de exemplar é do banco de dados, impulsionada por perda de conexão e outros fatores .
    Se na base de dados copi é se localiza a informação a ser aplicada na busca especificada, se avisa esta condição e não se permite a criação dos exemplar é . A mensagem inclui um link a fim de apresentar os exemplar é disponíveis na base de dados copies
  3. Se não localizar a informação na base de dados copies , proceda para a criaçãodos registros nesta base de dados, utilizando para isso os valor é que aparecem na nota de compra. ou valor correspondente ao número de exemplar é indica quantos registros se necessita criar na base de dados copies.

    Cada registro é atribuído um número sequencial correspondente ao número de inventário. Para este efeito, é necessário a presença do arquivo control_number.cn na pasta data da base de dados copies para armazenar ou último número atribuído ao inventário.

    Na FDT da base de dados copies se deve definir um campo do tipo AI (autoincrement) para indicar ou campo onde se vai armazenar ou número gerado.
