O bloqueio de registro acontece, em geral, no processo de entrada, como parte do mecanismo de proteção de atualização dos dados. Isto é, quando um usuário solicita modificar (editar) um registro, o mesmo é bloqueado para que não possa ser modificado por outro usuário simultaneamente. Quando o registro é atualizado, é desbloqueado e se torna disponível para outros usuários.

Se o ciclo de atualização não for completado, ou seja, o registro não for salvo e nem a edição cancelada (botões salvar ou cancelar da barra de ferramentas), o registro permanece bloqueado por um tempo (4 horas) para os demais usuários.

Existem duas maneiras de eliminar o bloqueio antes de esgotar o tempo acima referido:

  1. Se o mesmo operador que bloqueou o registro pede nova edição, será dado o direito à atualização do registro, podendo salvar ou cancelar a edição para liberar o registro.
  2. Outra forma de desbloquear o registro é utilizando a opção "Desbloquear registros" do menu Utilitários da barra de ferramentas. Somente o  Administrador do Sistema ou o Administrador da Base de Dados tem acesso a esta opção.

Menu de formatos de saída 

 Neste menu são apresentados os formatos de exibição (PFT), elaborados pelo administrador da base de dados. No final da lista são mostrados dois formatos adicionais:

  • Sem formato, o que indica que os campos serão apresentados em forma de tabela, identificados com as etiquetas (TAG's) definidas na FDT
  • Tudo, que apresenta o registro sem qualquer edição. É equivalente ao formato ALL das estruturas CDS/ISIS.
Menu de planilhas de  entrada Apresenta as planilhas de entrada de dados adicionais que foram definidas pelo administrador da base de dados. Quando se seleciona uma planilha de entrada de dados a partir da lista, ela fica ativa para os processos de criação e edição de registros.
