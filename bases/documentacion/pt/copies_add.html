Adicionar copias/exemplares a um objeto

 Os objetos para os quais se deseja criar um Número de Registro/Tombo, devem possuir um campo denominado "Número de controle", que deve estar definido na FDT como tipo Autoincrementar e estar indexado com o prefixo NC_ na FST. Este campo da FDT é utilizado como campo de relação entre o objeto e suas cópias/exemplares.

Os exemplares dos objetos das bases de dados confluem para uma base de dados denominada copies e será mantido um único Número de Registro para cada exemplar.

Para atribuição sequencial do número de controle em cada registro, é usado o arquivo control_number.cn localizado na pasta data da base de dados correspondente. Cada base de dados bibliográfica possui sua própria sequência de número de controle. Por esta razão, na base de dados copies é armazenado o nome da  base de dados relacionada com a cópia/exemplar para garantir a correta recuperação da informação bibliográfica, quando for necessário.

Quando é solicitada a inserção de novos exemplares, é exibido um formulário para entrada da informação correspondente.

Coloque na caixa do campo o número de cópias/exemplares que deseja adicionar ao objeto selecionado. Use o link Ver todas as cópias para exibir todas as cópias já adicionadas para este objeto.

Preencha todos os dados correspondentes e quando terminar clique em Atualizar.

Serão criados na base de dados copies os números de registros solicitados, de acordo com o valor do Número de cópias. A informação dos registros criados será tirada da planilha de entrada. O número de registro será adicionado automaticamente tomando como ponto de partida o valor armazenado no arquivo control_number.cn, armazenado na pasta bases/copies/data.

Use a opção Restabelecer o número de controle/Registro do menu principal de catalogação, se por alguma razão é necessário restaurar o próximo número de registro a ser atribuído às cópias, ou aos objetos de uma base de dados em particular.
