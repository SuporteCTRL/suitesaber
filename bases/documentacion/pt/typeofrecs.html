Tipos de registo

Se a estrutura da base de dados inclui diferentes tipos de registo, esta situação deve ser estruturada no ABCD.

O ABCD define duas formas diferentes de funcionamento com os tipos de registo:

  1. Para estruturas MARC
  2. Para estruturas diferentes de MARC

Tipos de Registo (estruturas Marc)

Esta opção ajuda a definir os tipos de registo que vão funcionar na base de dados quando a mesma utiliza o formato MARC. O ABCD identifica que uma base de dados está estruturada no formato MARC quando na tabela de definição de campos foi incluído o campo tipo LEADER (LDR). A estrutura do campo Leader encontra-se num ficheiro denominado leader.fdt, situado no directório def da base de dados, e que identifica os seguintes campos:

3005 Estado do registo
3006 Tipo de registo
3007 Nivel bibliográfico
3017 Nivel de codificação
3018 Descrição da forma de catalogação

O tipo de registo obtém-se a partir do campo 3006 do Leader e ajuda a identificar o formato do campo fixo a ser utilizado na descrição bibliográfica do título.

O formato do campo fixo associado a cada tipo de registo fica armazenado numa estrutura similar à da FDT e aparece no procedimento de entrada dependendo do tipo de registo seleccionado no Leader.

Para definir tipos de registo nas estruturas MARC, proceda como se segue:

  1. Active a opção Tipos de registo: marc no módulo de actualização das definições de bases de dados
  2. Coloque na caixa identificada como etiqueta1 a etiqueta do campo do Leader no qual se armazena o tipo de registo (3006). Deixe a caixa correspondente à etiqueta2 em branco
  3. Na coluna Folha de recolha de dados coloque o nome da estrutura que contém a descrição do campo correspondente ao tipo de registo que está a descrever. Use o link edit para editar ou criar a estrutura correspondente. Este link abrirá o editor de FDT dado que o campo fixo se mantém numa estrutura similar à da Tabela de Definição de Campos da base de dados
  4. Coloque na coluna correspondente à Etiqueta 1 Valor o valor que pode ter o campo 3006 para lhe associar esse tipo de registo. Deixe a coluna Etiqueta 2 Valor em branco
  5. Coloque na coluna Tipo de registo, Descrição a descrição do tipo de registo, tal como aparecerá no momento de criação para selecção do tipo correspondente 
  6. Repita os pasos 2-6 até esgotar os tipos de registo que deseja definir

Quando terminar clique no botão Actualizar

Os tipos de registo definidos serão armazenados no directório def da base de dados, sob o idioma activo no momento da criação, com o nome typeofrecord.tab

Tipos de registo (estruturas diferentes de MARC)

Esta opção ajuda a definir os tipos de registo que vão funcionar na base de dados quando a mesma não utiliza o formato MARC.

Para esse efeito é preciso definir o tipo de registo e atribuir que campos da base de dados (máximo dois) armazenam os valores a ser utilizados para identificar o tipo que corresponde ao registo em processo de criação ou edição.

Para definir os tipos de registo é necessário 2 componentes:

  1. Identificar o campo (etiqueta) da base de dados a partir da qual se vai obter (ou armazenar no caso de um novo registo) o tipo de registo correspondente à informação que se está registando. A versão actual do ABCD permite definir até dois campos a partir de cujo contéudo se determinará o tipo de registos e, consequentemente, a  folha de recolha a ser utilizada para a entrada ou edição do registo
  2. Descrever os tipos de registo tendo em conta os seguintes parâmetros:
    • Nome da tabela de definição (FMT) que contém os elementos a ser utilizados para construir o formulário de entrada para recolha dos dados
    • Valor que têm os campos identificados em (1) para que sejam considerados pertencentes a esse tipo de registo
    • Descrição do tipo de registo, a qual aparecerá ao operador no momento de entrada para que proceda à selecção do tipo correspondente

A fim de manter a consistência na parametrização do ABCD, os campos fixos definem-se através de uma FMT que contém a lista dos campos (etiquetas) da  FDT da base de dados que são necessários para construir o formato de entrada. Como só se mantém a etiqueta dos campos, estes herdam automaticamente as características definidas na FDT da base de dados.

Para atribuir os tipos de registo é preciso definir previamente as folhas de recolha que estarão vinculadas a cada tipo (ver Folhas de Recolha de Datos (FMT).

Se vai definir tipos de registo para uma base de dados que não segue a norma MARC, faça o seguinte:

  1. Coloque nas caixas identificadas como etiqueta1 e etiqueta2 as etiquetas dos campos que vão ser utilizados para identificar o tipo de registo. A segunda etiqueta é opcional
  2. Seleccione na coluna Folha de recolha a folha de recolha que vai utilizar para entrar o tipo de registo. Esta folha de recolha deve definir-se previamente com a opção Criar/Editar folhas de recolha
  3. Coloque o valor que terá o campo identificado em etiqueta1 valor para identificar o tipo de registo
  4. Coloque o valor que terá o campo identificado na etiqueta2 valor para identificar o tipo de registo (opcional)
  5. Coloque na coluna Tipo de registo, Descrição a descrição do tipo de registo, tal como aparecerá no momento de criação para selecção do tipo correspondente
  6. Repita os passos 2-5 tantas vezes quantos os tipos de registo que existirem na base de dados
  7. Quando terminar, clique em Actualizar

Os tipos de registo definidos ficarão guardados no directório def da base de dados, sob o idioma activo no momento da criação, com o nome typeofrecord.tab
