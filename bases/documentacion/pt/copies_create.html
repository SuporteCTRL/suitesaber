1. Apresenta informações do objeto obtidas a partir da base de dados do catálogo. Para determinar o prefixo a ser aplicado nesta busca leia o arquivo FDT da base de dados catalográfica e localize o campo tipo AI, o qual deve indicar o prefixo utilizado na indexação do número de controle do objeto. O valor do prefixo anexado à identificação do objeto, colocado na ordem de compra, provê a expressão de pesquisa a ser aplicada para a localização do objeto.

  2. Faz uma pesquisa na base de dados copies para checar se os objetos da presente nota de compra foram atualizados. Para isso constrói uma expressão de pesquisa com os seguintes campos:

    ORDER_order_suggestion_bidding

    onde order=número da ordem de compra, suggestion=número da sugestão, bidding=número da licitação.

    Esta pesquisa vai tentar evitar a duplicação na base de dados copies, por motivos de perda de conexão e outros fatores .
    Se na base de dados copies for localizada a informação a ser aplicada na busca especificada, é feito um aviso desta condição e não permite a criação dos registros na base copies. A mensagem inclui um link a fim de apresentar os exemplares disponíveis na base de dados copies

  3. Se não localizar a informação na base de dados copies, procede-se a criação dos registros nesta base de dados, utilizando para isso os valores que aparecem na ordem de compra. O valor correspondente ao número de exemplares indica quantos registros devem ser criadosr na base de dados copies.

    A cada registro é atribuído um número sequencial correspondente ao Número de Registro/Tombo. Para este efeito, é necessário a presença do arquivo control_number.cn na pasta data da base de dados copies para armazenar o último número atribuído.

    Na FDT da base de dados copies deve-se definir um campo do tipo AI (Autoincrementar) para indicar o campo onde será armazenado o número gerado.

    As informações mínimas que a base dados copies contém são as seguintes :

    4.  Tag Descrição
    1 Número de identificação do objeto
    10 Base de dados onde o objeto está catalogado
    20 Situação do exemplar
    30 Número de Registro/Tombo (autoincremento)
    40 Tipo de objeto
    50 Volume/Tomo
    60 Volume/Parte
    80 Data de aquisição
    90 Preço
    100 Número da ordem de compra

     

  4. Após a cópia do objeto criado e atribuído ou número de inventário, habilitado ou botão para a vão ar na criação dos correspondent é registros não banco de dados objetos de empréstimo (loanobjects).
