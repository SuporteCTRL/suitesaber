Resultado de la creación de una nueva base de datos

Al llegar a este procedimiento se crea la estructura para almacenar la nueva base de datos en la carpeta bases.

Las carpetas que se crean en este proceso son las siguientes:

  • bases/dbn/ayudas
  • bases/dbn/data
  • bases/dbn/def
  • bases/dbn/pfts
  • bases/dbn/cnv

Bajo cada una de estas carpetas, exceptuando las carpetas data y cnv, se crean tantas subcarpetas como lenguajes estén habilitados. La lista de lenguajes se obtiene del archivo  lang.tab ubicado en la carpeta bases.

Adicionalmente se crea el archivo dbn.par en la carpeta bases/par y se actualiza el archivo bases.dat agregando la base de datos ya creada.

Una vez finalizado el proceso de creación, debe asignar los operadores que pueden gestionar la nueva base de datos, ya que otra forma sólo estará disponible para el Administrador del sistema

Para llenar la base de datos debe obtener un archivo ISO con los datos para importarlo en la nueva estructura.

Si el archivo viene de WinIsis o de IsisDos, será necesario convertir el juego de caracteres de ASCII a ANSI. Esta tarea puede ser completada fácilmente utilizando el programa MX de la librería de funciones CISIS.

Para ello,  en una ventana DOS, ejecute el MX desde la carpeta donde está ubicada la base de datos que desea convertir utilizando la siguiente línea de comandos:

mx dbn  iso=dbn.iso convert=ansi -all now tell=100

El archivo iso obtenido debe ser subido al servidor a la carpeta data de la base de datos en proceso de creación. Para incorporar los registros use también el programa MX, con la siguiente línea de comandos:

mx iso=dbn.iso create=dbn -all now tell=100.

Luego de creada la base de datos, ejecute el MX nuevamente para generar la lista invertida, utilizando para ello la siguiente línea de comandos:

mx dbn fst=@dbn.fst uctab=ansi actab=ansi fullinv=dbn  -all now tell=100

Nota: Los comandos del mx suministrados aquí se dan como orientación. Para mayor información debe consultar el manual de referencia de la librería CISIS.

Si la cantidad de registros a importar no es muy grande (por ejemplo, menos de 2000 registros)  puede utilizar las opciones Utilitarios de la barra de herramientas del procedimiento de Entrada de datos para importar registros y generar la lista invertida.

Como el tiempo asignado por el servidor Web a cada proceso depende de cada instalación, el número de registros aquí mencionado puede variar, por lo que se recomienda, en lo posible usar el MX para estos propósitos
